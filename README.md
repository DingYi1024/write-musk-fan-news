# write-musk-fan-news

> 把马斯克相关新闻、科技新闻和已有账号内容，重构成“科技新闻外壳 + 马斯克人物叙事 + 可争论结论”的原创中文视频文案。

## 这不是一个火箭科普 Skill

它抽象的是一种“马斯克粉丝型科技新闻账号”的内容机制：

- 新闻只是入口，马斯克才是稳定主角。
- 火箭、星舰、机器人、AI、特斯拉和商业新闻都可以成为人物证据。
- 科技奇观负责吸引观看，传统方案与反常决策负责制造冲突。
- 结尾最终落回马斯克的能力、方法、使命或未来图景。
- 争议可以来自解释和立场，但事实、数字和引语不能编造。

这个 Skill 源于对抖音同类账号叙事方法的拆解，但不会复制对标账号的原句、口头禅、段落顺序或独特比喻。它做的是原创重构，不是逐句洗稿。

## 快速开始

安装后，直接把新闻、链接、事实材料或待改写文案交给 Agent：

```text
$write-musk-fan-news
把下面这条星舰新闻写成一篇 2 分钟的视频号口播稿。
核心受众是喜欢马斯克、但不懂航天技术的人。
新闻材料：……
```

在支持斜杠命令的 Agent 中也可以使用：

```text
/write-musk-fan-news 把这条马斯克新闻写成小红书 90 秒视频稿：……
```

改写其他账号内容时：

```text
$write-musk-fan-news
保留下面原稿中可核实的事实，但重新选择角度、重排结构并重写成原创稿。
不要沿用原文开头、论点顺序和结尾。
原稿：……
```

## 能力一览

| 任务 | Skill 会做什么 | 默认产出 |
| --- | --- | --- |
| 写马斯克新闻 | 从新闻事实中提取画面、数字、冲突和人物落点 | 标题、开头、完整口播稿 |
| 写科技或商业新闻 | 判断新闻能否自然连接马斯克及其公司 | 马斯克中心的原创叙事 |
| 重写对标账号文案 | 事实解包、观点清零、重选角度、重排因果 | 与来源结构明显不同的新稿 |
| 重做自己的旧稿 | 保留有效事实，替换开头、冲突和结论 | 新版本口播稿 |
| 一稿多发 | 根据不同平台调整信息顺序、长度和争议强度 | 母稿与平台适配方案 |
| 诊断现有稿件 | 检查开头、人物中心、证据、争议与原创度 | 风险点和修改方向 |

默认完整输出包括：

1. 一句话选题判断
2. 事实核与争议点
3. 5 个标题与 3 个五秒开头
4. 可直接朗读的完整口播稿
5. 分段素材功能提示
6. 事实、推断与高风险表述检查

如果只需要成稿，可以直接说“只要文案”。

## 五种写作引擎

| 引擎 | 适用内容 | 核心推进方式 |
| --- | --- | --- |
| 奇观纪实型 | 发射、回收、爆炸、工厂、罕见画面 | 先展示结果，再解释过程，最后落回人物能力 |
| 反常解释型 | 不锈钢、复用、成本、工程选择 | 旧方案很贵或很慢，马斯克选择反常路径并被结果证明 |
| 人物神话型 | 团队、管理、财富、经历、公开言论 | 用具体决策证明人物特质，不空喊伟大 |
| 现实连接型 | 上网、出行、工作、钱、本地目击 | 把遥远科技变成普通人身边的问题 |
| 事件追踪型 | 持续发生的测试、发布与竞争事件 | 拆成预告、变化、结果、细节和人物解释连续开发 |

## 支持的平台

- 视频号、快手：强化完整故事、现实影响和人物结论
- B站、YouTube 中文：增加背景、技术限制、来源和反方观点
- 小红书：突出工作、效率、财富、旅行和生活变化
- 今日头条、西瓜视频：强化时间线、搜索关键词和新闻背景
- X、Threads：重构为 6—10 条连续短帖

Skill 会先完成一篇事实完整的中文母稿，再按平台重新组织，而不是机械删字。

下载仓库后，把以下目录复制或链接到 Agent 的 Skills 目录：

## 工作原理

```text
新闻、链接或已有文案
        ↓
提取可验证事实核
        ↓
选择一个主写作引擎
        ↓
建立旧做法与新做法的冲突
        ↓
用画面、数字、成本或结果证明
        ↓
连接普通人的现实利益
        ↓
给出偏向马斯克但可以争论的解释
        ↓
输出平台可用的原创文案
```

## 原创与事实边界

Skill 遵循四条底线：

- 模仿叙事机制，不模仿具体表达。
- 不逐句替换同义词，不保留来源的段落顺序。
- 近期新闻、数字和直接引语优先核对权威来源。
- 允许评论区反驳观点，不让核心事实轻易被证伪。

争议公式是：

> 可验证事实 + 偏向人物的解释 + 有倾向的对比 + 可争论的结论
