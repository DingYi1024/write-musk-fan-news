# 后台来源与前台口播分离系统

## 核心原则

> 后台查来源，前台讲事实；不做媒体摘要，也不把别人的独家调查洗成自己的发现。

来源解决“能不能相信”，口播解决“观众愿不愿意听”。两者必须分层处理。

- **后台证据层**：记录网页、文件、发布时间、关键数字、来源等级和仍需复核的字段。
- **前台口播层**：直接讲画面、动作、数字、反差和马斯克中心解释，默认不朗读新闻媒体名称。

不在口播中说媒体名，不代表自动合规。如果事实只能由某家媒体的独家调查、匿名消息或专有估算支持，不能删掉媒体名后冒充自己的事实；默认更换选题，或找到公开一手材料后再写。

## 对标账号复核结论

2026年7月17日重新抽样“芬达说”主页近20条作品标题，并查看两条视频的页面章节摘要：

- 标题普遍采用“马斯克/SpaceX + 直接事实 + 数字或反差 + 人物判断”。
- 口播摘要直接讲火箭画面、员工数量、管理方式和人物结论，没有连续播报新闻机构名称。
- 页面强调“个人观点，仅供参考”，叙事者站在解释者位置，而不是媒体转述者位置。
- 当来源本身具有画面或人物价值时，会写“SpaceX直播”“SpaceX公布”“马斯克表示”；这些是事件主体的一手公开动作，不是第三方新闻背书。

因此，风格目标不是“假装没有来源”，而是：

> 一手主体可以进入故事，第三方媒体留在后台核验。

## 四级来源路由

| 类型 | 例子 | 前台处理 | 默认决策 |
| --- | --- | --- | --- |
| P1 一手公开材料 | SpaceX直播、公司公告、监管文件、法院文件、马斯克公开帖文或采访 | 可直接讲事实；名称只在有叙事价值时出现一次 | 优先使用 |
| P2 可独立验证的公开事实 | 多个公开记录一致的日期、数字、产品参数、历史节点 | 直接讲事实，不朗读媒体名单 | 可以使用 |
| M1 单一媒体独家调查 | 独家采访、内部文件、匿名消息、记者现场调查 | 不能去掉归因后伪装成自有事实 | 默认不用作极简稿主轴 |
| M2 媒体或作者的专有估算 | 根据设备资料推算排放、成本、人员或概率 | 找到原始数据并自行说明方法，否则不使用 | 默认删除 |

## 选题准入流程

### 1. 先问事实从哪里来

- 能否从公司直播、公开文件、当事人原话、监管记录或现场画面直接确认？
- 是否存在至少两个彼此独立的公开来源？
- 关键数字是不是某家媒体自己的计算？
- 如果删掉媒体名，观众是否会误以为这是账号自己的调查结果？

### 2. 再决定能不能写

- **P1/P2为主**：正常写作。
- **P1不足、M1为主**：默认换题；用户明确要求时，保留必要归因，并输出风险提示。
- **M2为主**：找到原始数据并重新计算；做不到就删除数字或放弃选题。
- **只有匿名消息**：不写成事实，不做标题主张。

### 3. 最后才做口播化

把第三方媒体留下的事实线索，替换为可验证的一手对象：

- `路透社报道，SpaceX将在……` → `SpaceX公布的任务安排显示……`
- `美联社称马斯克表示……` → 找到马斯克原话后写 `马斯克随后表示……`
- `媒体根据资料估算排放……` → 找到许可/排放文件并说明数字，找不到则删除。

不能完成替换时，不得机械删除“路透社”“美联社”后继续使用其独家内容。

## 前台口播规则

默认正式口播不得出现：

- “路透社报道/调查发现/根据资料估算”
- “美联社报道”
- “彭博社消息”
- “据外媒/据多家媒体/有媒体称”
- 同一篇中连续出现两个以上第三方新闻机构

允许出现：

- `SpaceX直播画面显示……`
- `特斯拉提交的文件里有一个数字……`
- `马斯克随后公开表示……`
- `当地公开许可文件显示……`

即使允许，也只在来源本身能提供画面、人物动作或关键证据时使用；否则直接讲事实。

## 后台来源卡

写作前在内部保留：

```text
事实：
P1一手材料：
P2交叉记录：
是否来自单一媒体独家：是/否
是否含媒体专有估算：是/否
可直接进入口播的事实：
必须删除或保留归因的内容：
截止日期：YYYY-MM-DD
```

用户说“只要文案”时不展示来源卡。用户要求事实核、发布备注或合规检查时，再单独输出；不要把来源卡混进口播稿。

## 截图案例诊断

问题稿连续出现：

- “路透社调查发现”
- “路透社根据设备资料估算”
- “美联社报道”

这会产生三种问题：

1. 叙事者像新闻汇编者，没有自己的账号声音。
2. 事实主轴依赖媒体独家材料，删掉署名又可能形成错误归因。
3. 观众记住的是媒体，不是“马斯克 + 反常选择 + 自媒体判断”。

正确处理不是简单改成“公开资料显示”，而是先审查题目：

- 59台燃气轮机、至少57台部署、排放估算若只能由单一媒体调查支持，则默认不写这篇。
- 若能找到公开许可、监管文件或xAI/设备方资料，再用这些一手事实重建故事。
- 环保组织观点、监管判断和诉讼状态分别归因，不能合并成账号自己的结论。

## 原创与合规边界

- 不复制媒体原文、段落结构、独特比喻或专有图表。
- 转述他人信息不自动等于原创调查。
- 转载内容依法尊重著作权和可追溯来源要求；如果必须公开标注来源，就不要为了“像自媒体”而故意隐藏。
- 公众账号从事互联网新闻信息服务可能涉及许可要求；是否构成相关服务取决于实际内容、账号行为和适用规则，不能靠改写措辞规避。
- 优先做“公开事实 + 人物解释 + 科技评论”，避免冒充新闻采编、现场调查或独家爆料。
