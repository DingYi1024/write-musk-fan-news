# 平台适配

先完成一篇事实完整的中文母稿，再按平台裁剪。不要用同一开头和同一长度机械分发。

## 默认母稿

- 时长：约2分钟
- 字数：约500—750字，服从实际语速
- 结构：强事实开头、一个主要冲突、三组证据、人物落点
- 画面：混剪、新闻画面、官方素材、示意动画和数据卡

## 视频号 / 快手

- 建议时长：1—3分钟
- 强化完整故事、创业精神、现实影响和人物结论
- 少用内部型号堆砌；专业名词出现后立即翻译成日常语言
- 结尾形成明确态度，方便转发给熟人

## B站 / YouTube中文

- 建议时长：4—8分钟
- 增加来源、背景、技术限制和一个反方观点
- 保留鲜明立场，但先替反对者回答最强质疑
- 章节结构清楚：事件、旧方案、马斯克方案、证据、限制、未来

## 小红书

- 视频建议60—120秒；图文建议6—10页
- 优先普通人、工作、效率、财富、旅行和生活变化
- 标题少用连续感叹，突出“这件事和我有什么关系”
- 图文每页只承担一个事实或判断

## 今日头条 / 西瓜视频

- 建议2—5分钟
- 强化新闻时间线、事件背景和可搜索关键词
- 开头先给结果，正文补充来龙去脉
- 标题保留人物名、公司名、事件名和关键数字

## X / Threads / 文字平台

- 改成6—10条连续短帖
- 首帖只放结论和最强事实
- 中间每帖一个证据
- 最后一帖给人物解释和争议问题

## 一稿多发输出格式

用户要求多平台版本时，输出：

1. 母稿
2. 各平台标题
3. 各平台专属开头
4. 需要删除、增加或前移的段落
5. 字数与预计时长

不要为每个平台简单删字；必须调整信息顺序和争议强度。

## 鸡汤强度适配

- **视频号 / 快手**：可使用一级或二级鸡汤，突出创业、责任、家庭可理解的现实意义；结尾不超过全文15%。
- **B站 / YouTube中文**：先补来源、代价和反方观点，再做方法论升华；不把争议管理方式写成普遍真理。
- **小红书**：把宏大使命转成一个普通人可执行的小动作，避免居高临下地教育读者。
- **今日头条 / 西瓜视频**：人物故事必须保留时间线和事件名，鸡汤不能替代新闻信息。
- **X / Threads**：用一条原创转述承担升华，不连续发布语录卡式句子。
