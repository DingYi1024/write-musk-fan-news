# 风格系统

## 核心定义

把世界新闻组织成马斯克中心叙事：新闻是入口，科技奇观是证据，争议是传播，人物英雄化是最终交付。需要鸡汤时，必须从人物已经付出的代价和已经采取的行动中提炼，不额外粘贴空洞正能量。

账号样本显示，内容不平均覆盖马斯克旗下公司，而是高度使用SpaceX、星舰和火箭。这些素材具备宏大画面、明确结果和低理解门槛，最容易证明马斯克“敢做、能做、做得快”。

## 马斯克中心度：事件只能做人物证据

“马斯克中心”不是在开头和结尾各写一次名字，而是让整篇稿子的因果主语始终是马斯克。新闻对象负责回答“他为什么这样判断”，不能取代他成为正文主角。

写作前先把选题改写成一个人物命题：

> 这件事证明了马斯克哪一个可观察的行为、判断、能力或一贯模式？

若答案只是“某产品很厉害，所以马斯克点赞”，题目还没有完成。继续追问：他看见了什么、公开做了什么、这与他以往或同行的做法有何不同、观众最终为什么要重新认识马斯克。

### 70/30叙事预算

- 约70%的叙事功能服务马斯克：他的动作、判断、选择、对比、历史模式、影响或人物结论。
- 新闻对象、产品参数和第三方背景合计不超过约30%，只保留足以证明马斯克判断的2—3个事实。
- 这不是机械统计“马斯克”出现次数。同一句里即使写了马斯克，若主要在介绍别人的产品，仍计入事件背景。
- 不得连续超过两句只讲其他人物、公司、产品或技术；下一句必须回到马斯克的动作、判断或人物模式。
- 每个段落都应能用“马斯克……”概括其叙事作用。无法概括的背景段落，删除、压缩或改成人物证据。

反应型新闻采用更严格的预算。凡是马斯克点赞、回复、转发、称赞或批评外部对象，产品背景最多两句，并按`M人物—E证据—M解释`交替推进。完稿后逐句标记：

- `M`：马斯克的动作、判断、历史行为、影响或人物解释。
- `E`：其他产品、公司、人物、参数、流程、时间线或行业前景。
- `X`：纯转场；若转场实际在概括产品意义，仍标E。

M类句至少为E类句的两倍，且E后立即回到M。句子出现“马斯克”三个字但主要在介绍对方，仍标E。若不达标，删除技术细节或补入可核实的马斯克既往行为后重写。

使用删除测试防止伪M句：删掉“马斯克认为/马斯克看到”后，剩余内容若仍在完整介绍产品能力、行业趋势或对方未来，这句仍是E。例如“他看到AI正在接手复杂任务”讲的是AI趋势，不是马斯克人物证据。

反应型稿件不要自由铺陈，先使用以下固定句位：2句本次马斯克动作，1句外部结果，2句马斯克解释，2句马斯克既往行为与共同标准，最多1句外部边界，3—4句对比、反方和人物结论。缺少本次反应之外的马斯克事实时，缩短或换题，不用产品细节补时长。

### 人物回环结构

涉及外部产品、同行或中国科技公司的新闻，默认按以下顺序写：

1. **人物钩子**：马斯克做了什么反常动作，这个动作为什么比事件本身更值得讲。
2. **动作现场**：给出他的原话、点赞、转发、决定或公开回应。
3. **最小证据**：只用1—2个事实解释被他看见的对象为什么值得回应；每个事实后立即回到人物解释。
4. **人物对比**：用已核实的同行公开行为、行业惯性或马斯克过往行为，反衬这次选择。
5. **一贯模式**：说明公平评价、工程判断、开放竞争、求真或长期主义中的一个主题；不要同时贴多个品质标签。
6. **人物落点**：结尾回答“这件事让人重新认识了马斯克什么”，不总结对方产品前景。

对比只写可验证的行为。没有证据时，不写“其他AI创始人都在贬低中国AI”“整个西方都沉默了”等群体判断；可改成行业中确实存在的竞争姿态，或选取一个有公开记录的具体对比对象。

### Kimi点赞案例

跑偏结构：马斯克留下一句评价 → 连续介绍Kimi参数、芯片设计流程、发布时间和行业排名 → 用“他看见AI正在变化”伪装成人物解释 → 结尾再提马斯克。

正确结构：马斯克公开肯定中国AI → 用一个事实说明这份肯定并非客套 → 立刻解释这反映了马斯克怎样的判断 → 再用一个事实补强 → 加入一个可核实的马斯克既往行为或具体同行对比 → 落到他的判断、公平或自信。没有历史证据时，不使用“一贯”“从来”。

标题同样以人物动作为主：优先“马斯克为何只用一个词肯定中国AI”，不要把“中国AI 48小时完成芯片设计验证”写成真正标题主语。

合格的12句骨架如下。它示范叙事角色，不要求复用措辞：

1. M：马斯克夸中国AI，最反常的不是一个英文词，而是他作为直接竞争者仍公开承认结果。
2. M：交代他在Kimi相关讨论中回复`Impressive`。
3. E：只用一句说Kimi连续48小时完成芯片设计、优化和验证的概念演示；不再列参数、工艺、吞吐或发布日期。
4. M：解释马斯克为什么更容易注意“长时间完成真实工程”，而不是参数榜单。
5. M：调用A01，写他曾公开修正对比亚迪的旧判断，承认对方已经很有竞争力。
6. M：解释这证明他至少会根据新的产品结果更新公开评价。
7. M：调用A02，写SpaceX负责人也曾公开承认中国航天能力被低估。
8. M：解释本次行为与过去行为的共同点：先看做成了什么，再下判断。
9. M：强调他自己经营xAI，是竞争者而非局外评论员，因此公开肯定更有反差。
10. M：承认一个词不足以证明完整人格，也可能只是即时反应。
11. M：但一次次可核实的公开行为，足以支持“他会承认已经发生的进步”这一有限结论。
12. M：结尾只落马斯克的判断标准，不总结Kimi未来、AI行业前景或工程师工作流。

若成稿在第3句后继续解释45纳米、100MHz、8700 Token、模型参数或芯片前景，即使后面调用A01/A02，也仍判定失败并重写。

## 品牌声纹：固定口头禅 + 变化钩子

所有正式口播稿统一以“你肯定不敢相信，”开头。固定的只有前七个字，后半句必须根据当篇事实重新创作。

开头公式：

> 你肯定不敢相信，主体 + 表面结果不是A + 真正关键是B。

六种可用反转：

| 反转类型 | 后半句任务 | 示例骨架 |
| --- | --- | --- |
| 结果反转 | 观众以为失败，真正价值在另一结果 | 最值钱的不是成功，而是系统拒绝了错误 |
| 成功反转 | 表面成功不如隐藏能力重要 | 真正赢的不是升空，而是下一次成本 |
| 身份反转 | 打破对人物或搭档的固有认知 | 最早的搭档不是大佬，而是家人 |
| 成本反转 | 大项目与低成本选择冲突 | 最昂贵的目标，偏偏用最便宜的材料 |
| 关系反转 | 关系中的角色与公众印象相反 | 最理解他的人，后来却成了对手 |
| 未来反转 | 当前小事件指向更大变化 | 今天停下的一秒，可能决定下一代火箭 |

钩子纪律：

- 第一句18—45个汉字；口头禅与钩子不分句。
- 第一句出现马斯克或明确主体，并至少给出一个具体动作、数字、关系或结果。
- 日期、地点、媒体和消息来源延后到第二或第三句。
- 不使用问候、铺垫、百科定义、会议名称或“今天发生大事”开头。
- 不虚构“所有人震惊”“专家沉默”等群体反应。
- 删除“你肯定不敢相信”后，剩余部分仍应是一句有信息量的钩子；否则重写。

同一事件对比：

- 新闻播报：`7月16日，美国得州星际基地。124米高的星舰已经点火。`
- 贴标签式伪改：`你肯定不敢相信。7月16日，美国得州星际基地……`
- 品牌化表达：`你肯定不敢相信，马斯克这次最值钱的不是点火，而是火箭在最后1秒拒绝起飞。`

固定口头禅负责一秒识别，变化钩子负责当篇留存。不能只完成前者。

## 去媒体摘要感

对标账号的前台语言是“直接事实 + 数字反差 + 人物解释”，不是“媒体A报道 + 媒体B估算 + 媒体C补充”。

- 默认不在口播中点路透社、美联社、彭博社等第三方媒体名称。
- 一手主体具有故事价值时，可以说“SpaceX直播”“马斯克表示”“公司文件显示”。
- 来源只在后台核验，不把来源列表写进口播。
- 如果事实只来自媒体独家调查，不能简单删掉媒体名；默认换题或先找一手文件。

详细准入和合规规则见 [source-narration-system.md](source-narration-system.md)。

## 固定世界观

| 角色 | 常见功能 |
|---|---|
| 马斯克 | 反常决策、低成本、快速迭代、开放、长期使命 |
| 传统公司 | 慢、贵、流程复杂、害怕失败 |
| 专家 | 代表旧经验，可成为反差对象 |
| 竞争者 | 用来证明马斯克领先，或制造“终于有对手” |
| 普通人 | 把遥远科技连接到上网、出行、工作、钱和未来 |
| 评论区 | 支持者确认信仰，反对者纠错，路人围观 |

不必每次攻击传统方案，但必须存在一个“旧理解”和一个“新事实”。

## 六种文案引擎

### 1. 奇观纪实型

适合：罕见追踪画面、发射、回收、爆炸、工厂、现场全过程。

结构：

1. 第一秒展示结果或最强画面。
2. 说明画面属于谁、发生在哪里。
3. 按时间顺序解释关键动作。
4. 在最难理解处补一句技术解释。
5. 用成功结果完成情绪释放。
6. 结尾归因马斯克或SpaceX的能力。

语气：少讲道理，多描述正在发生的事。画面本身承担赞美。

### 2. 反常解释型

适合：不锈钢、重复使用、员工数量、生产方式、工程选择。

结构：

1. 用生活物品或常识制造反差。
2. 建立传统高价/高端方案。
3. 说明传统方案在真实任务中的困境。
4. 给出马斯克的反常选择。
5. 用成本、时间、耐久性、规模或结果验证。
6. 从局部选择升到整体效率。
7. 最终连接火星、机器人、全球通信等长期目标。

核心句法不是“马斯克很厉害”，而是“别人为什么没这样做，而他为什么敢”。

### 3. 人物神话型

适合：年轻工程师、管理、财富、家庭、公开言论、个人习惯。

使用3—4个具体决策证明一种人格特质。每个特质都配事实或案例，不列空洞品质清单。

常见落点：

- 抛弃历史经验
- 缩短决策链
- 允许快速失败
- 用使命吸引人才
- 把昂贵工程改造成流水线产品

### 4. 现实连接型

适合：国内目击、免费上网、普通人旅行、生活成本、工作替代。

优先顺序：本地地点 > 普通人体验 > 明确数字 > 中外反差 > 人物解释。

这类内容的转发动力通常高于纯硬件新闻。不要只问“技术多强”，要问“它为什么已经来到我们身边”。

### 5. 事件追踪型

同一事件可以连续开发：

1. 时间与预告
2. 与上一版本的变化
3. 当天结果
4. 全过程剪辑
5. 单个罕见细节
6. 马斯克决策解释
7. 对普通人的影响

预告数据通常弱于结果和罕见画面。高价值素材出现时，允许同日发布多个不同角度，不用担心题材重复。

### 6. 硬核鸡汤型

适合：低谷、失败、孤注一掷、使命、方法、工作和人生选择。

结构：

1. 从最低点、危险时刻或反常选择开场。
2. 交代可验证的限制、代价和失败概率。
3. 展示人物采取的具体动作。
4. 给出结果，也承认团队、运气和副作用。
5. 只提炼一个普通人可带走的方法。

鸡汤不能替代事实。普通新闻只在结尾使用1—2句；完整鸡汤稿至少需要三个具体事实。详细规则见 [musk-wisdom-system.md](musk-wisdom-system.md)。

## 标题系统

标题一般控制在20—36个中文字符，采用“人物/主体 + 事实 + 情绪解释”的三段式。

常用结构：

- 马斯克做了什么 + 极端数字 + 谁被迫重新判断
- 普通人已感受到什么 + 国内/传统方案仍如何 + 为什么
- 罕见画面 + 具体结果 + 这意味着什么
- 别人使用旧方案 + 马斯克改用什么 + 结果如何
- 一次事件 + 多个纪录 + 马斯克下一步做什么

可使用感叹号、问号、数字和对比，但标题必须含真实信息。不要凭空制造“全网刷屏”“全球沸腾”“专家集体沉默”。

## 语言节奏

- 句长以10—22字为主，长句拆开。
- 每段先事实，后解释，再落态度。
- 使用具体动词：发射、回收、拆除、公开、降落、缩短、节省、复用。
- 少用抽象名词：赋能、格局、生态、颠覆性价值。
- “竟、只有、直接、终于、为何”可用，但不要每段都用。
- 不连续使用三个形容词。
- 不连续使用三个排比句。
- 不使用“请记住”“真相是”“值得我们深思”等AI腔。
- 不使用“只要努力就一定成功”等无条件承诺；先写困难、选择和代价，再写意义。

## 三层结论

根据证据强度选择结尾层级：

1. **事实层**：这次测试证明某项能力已经实现。
2. **方法层**：结果来自低成本、快速迭代或工程选择。
3. **人物/未来层**：马斯克正在把某项昂贵能力变成规模化产品。

证据弱时停在事实层。证据充分时才升到人物或文明叙事。

## 争议设计

优先制造以下争议：

- 工程效率与流程安全
- 开放与保密
- 低成本与高可靠
- 快速失败与稳妥推进
- 个人英雄与团队贡献
- 国外进展与国内体验
- 科技未来与现实利益

最佳状态：支持者能转发，反对者能指出解释偏向，但双方都承认核心事件确实发生。
