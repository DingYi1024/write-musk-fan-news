# 马斯克反应型新闻专用系统

## 触发条件

新闻核心动作是马斯克对外部人物、产品、公司或技术进行点赞、回复、转发、称赞、批评或简短评价时，停止使用普通科技新闻结构，改用本系统。

目标不是解释对方有多强，而是解释：

> 马斯克为什么做出这个公开反应，这个行为与他过去哪些可核实的行为一致，它让观众重新认识了马斯克什么。

## 写前证据包

只准备四类材料：

1. `M-now`：马斯克本次原话或公开动作及语境。
2. `E-proof`：被评价对象唯一一个足以解释本次反应的结果事实。
3. `M-past`：至少一张 [musk-public-behavior-anchors.md](musk-public-behavior-anchors.md) 卡，或另一项可核实的马斯克既往行为。
4. `M-contrast`：马斯克的竞争身份、另一个既往行为，或一项具体同行公开行为。

没有`M-past`就不写90秒。缩成45—60秒，或换题；严禁增加产品参数补时长。

## 唯一允许的句位

先在内部逐句填标签，不得自由铺陈：

```text
M1 你肯定不敢相信，+ 马斯克本次反常动作
M2 本次原话/动作及其竞争语境
E1 对方唯一一个结果事实；一句说完
M3 这个结果为什么会进入马斯克的判断范围
M4 马斯克过去一次可核实的公开行为
M5 过去行为如何证明他会更新判断或承认结果
M6 第二项马斯克证据，或他作为直接竞争者的反差
M7 本次行为与过去行为的共同标准
M8 合理反方：一次互动可能只是即时反应、策略或礼貌
M9 即使接受反方，本次行为仍能支持的有限结论
M10 只回到马斯克的判断、方法或人物反差
```

只有当不写边界会造成事实误导时，才在E1后增加一句`E2`必要边界。除此之外不得增加E句。

## E句识别

以下内容都属于E，不能因句首出现马斯克就算M：

- 产品参数、模型能力、工艺、吞吐、发布日期和排名。
- 对方公司的发展史、团队和商业前景。
- “马斯克看到AI正在改变工程师工作”这类主要解释AI趋势的句子。
- “因为这说明……”后继续介绍被评价对象或行业变化的句子。

使用删除测试：删掉“马斯克认为/看到/注意到”，剩余部分仍是一句完整的产品或行业介绍，就是E。

## 成稿闸门

输出前在内部检查；任一项失败就重写，不得带病输出：

- 标题主语是马斯克的动作或判断。
- 第一、第二句都是M。
- 默认只有一个E句；必要边界存在时最多两个。
- 不出现E—E相邻。
- M句不少于8句，且至少包含本次反应之外的一项马斯克事实。
- “一贯、从来、总是”有两个以上马斯克公开行为支持，否则改成有限判断。
- 结尾三句不讨论产品未来、行业前景、就业变化或谁将获胜，只讨论马斯克。

## Kimi回归闸门

写“马斯克回复Kimi K3为`Impressive`”时：

- E1只保留“官方展示K3连续48小时完成芯片设计、优化和验证的概念案例”。
- 可说明这不是量产芯片，但只有在防止误导时才作为E2。
- 成稿禁止出现`2.8万亿`、`100万token`、`45纳米`、`100MHz`、`8700`、完整权重发布日期。
- M-past优先使用A01比亚迪判断更新；可再用A02中国航天或A05竞争者算力。
- 人物结论只写“会根据已经做出的结果更新或公开表达判断”，不能证明他永远公平，也不把一个词神化。

## 合格方向

观众看完应能复述：

> 马斯克作为竞争者仍公开承认结果；这与他后来改口承认比亚迪、肯定中国航天等行为形成同一判断模式。

若观众更容易复述Kimi参数、芯片工艺、Google算力规模或另一家公司前景，这篇稿就仍然跑偏。
