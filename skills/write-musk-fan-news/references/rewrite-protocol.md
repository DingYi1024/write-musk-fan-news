# 原创重构协议

## 适用范围

当输入是其他账号文案、视频字幕、文章、新闻稿或本账号旧内容时使用。目标不是“降低重复率”，而是基于同一事实重新做一次选题、判断和叙事。

## 六步重构

1. **事实解包**：只保留人物、时间、地点、数字、动作、结果、可靠引语。
2. **观点清零**：暂时删除来源的态度、比喻、情绪词和结论。
3. **重新选角度**：从奇观、反常决策、普通人影响、争议比较、人物方法中另选主轴。
4. **重排因果**：不要沿用来源的段落顺序。可从结果倒叙、从生活场景切入或从失败切入。
5. **重建表达**：重新选择比喻、例子、对手和结尾，不做逐句同义替换。
6. **独立校验**：用至少一个独立来源核对关键数字和引语；删除无法核实的细节。

## 相似性红线

- 不复用来源的标志性开头、口头禅、结尾或独特比喻。
- 除专有名词、正式引语和不可避免事实外，避免连续复用来源8个以上汉字。
- 不保留相同段落数量、相同论点顺序和相同例子组合。
- 不以“改写”“原创”为理由掩盖逐句洗稿。
- 若原内容本身只有一个事实点，回到原始新闻扩充，而不是硬改原文。

## 三种重构方式

### 同新闻重构

把原稿的“发生了什么”改成“为什么马斯克会这样做”或“它怎样影响普通人”。

### 跨人物套用

保留英雄叙事机制，但根据新人物真实特质重建：不能把所有人物都写成马斯克。先找此人的反常决策、长期目标和主要对手。

### 旧稿重做

保留已验证的事实，换开头、主冲突、证据顺序和结尾。若旧稿已有数据反馈，保留有效母题，删除无效表达。

## 输出检查

在最终答案中单列：

- 保留的事实核
- 新选择的叙事角度
- 与来源不同的结构变化
- 仍需核实的数据或引语

用户要求只看成稿时不展示这些分析，但仍在内部完成。
